Radical Portability

Redcap is built on top of Redbean, an Actually Portable Executable (APE). This means your entire website—server, database, admin panel, and content—is contained within a single file that runs natively on almost any operating system.

# Move your entire site with one command
cp mysite.com user@server:/var/www/

Bake to ZIP

Live changes to your SQLite database can be "baked" directly back into the executable's internal ZIP store.

Visual Builders

Define schemas and author content using a modern, reactive interface. No coding required.

Monaco IDE

A pro-grade layout editor with syntax highlighting and live side-by-side previewing.

Headless API

Automatically generated REST/JSON endpoints for all your content collections.

Multi-Language

Native i18n support for global reach with localized routing built-in.

Self-Healing

Automatic integrity checks and daily backups ensure your data remains robust and recoverable.

Cloud Deployment (Fly.io)

Redcap is container-ready. Deploy to Fly.io in seconds with this minimal Dockerfile:

FROM alpine
COPY redcap.com /redcap.com
CMD ["/redcap.com", "-p", "8080", "-*"]

Quick Start

1. Initialize your admin account:

./redcap.com -- --init-admin="mypassword"

2. Run the atomic server:

./redcap.com -p 8080 -*

3. Build your world at /admin.
